# AGENTS.md

Personal portfolio site for Johannes Fischer. Static React SPA deployed to GitHub Pages.

## Tech Stack

- **React 19** — functional components, hooks
- **TypeScript** — strict mode, ES2023 target
- **Vite** — build tool and dev server
- **Panda CSS** — atomic CSS-in-JS, design tokens, CVA variants
- **Framer Motion** — animations
- **Lucide React** — icons

## Commands

```bash
npm run dev        # dev server with HMR
npm run build      # production build → dist/
npm run preview    # preview production build
npm run lint       # ESLint on src/
npm run format     # Prettier on all files
npm run prepare    # regenerate Panda CSS types (panda codegen)
```

Run `npm run format && npm run lint` before committing.

No tests exist. `npm test` exits with error by design.

## Project Structure

```
src/
├── components/    # reusable UI (Button, Carousel, Header, Footer, …)
├── sections/      # page sections (Hero, Skills, Testimonials, Career)
├── styles/        # CVA variant definitions (button.ts)
├── App.tsx        # root, theme provider
├── ThemeContext.tsx
├── usePreferredColorScheme.ts
├── const.ts       # URLs and constants
└── main.tsx
styled-system/     # auto-generated by Panda CSS — do not edit
```

## Code Conventions

### Components

- Files: PascalCase (`Button.tsx`, `HeadingGroup.tsx`)
- Hooks/utils: camelCase (`usePreferredColorScheme.ts`)
- Constants: UPPER_SNAKE_CASE (`COLOR_SCHEMES`, `URLS`)
- Use `React.FunctionComponent<Props>` type for components
- Default exports for components

### Styling

All styling via Panda CSS — no traditional CSS files.

```typescript
import { css } from "../../styled-system/css";
import { grid } from "../../styled-system/patterns";

<div className={css({ bg: "primary", color: "neutral", p: 4 })} />
```

Responsive values use arrays: `{ p: [2, 4, 6] }` = mobile / tablet / desktop.

Button variants defined with `cva()` in `src/styles/button.ts`. Compose via `buttonStyles({ variant, size, disabled })`.

### Theme

Dark/light theme via `ThemeContext`. System preference detected by `usePreferredColorScheme` using `useSyncExternalStore`. Applied as `data-color-mode` attribute on root div.

### Data

No external API calls. Content is hardcoded in section components. Shared URLs/constants live in `src/const.ts`.

## TypeScript

Strict mode enabled. Key flags: `noUnusedLocals`, `noUnusedParameters`, `noUncheckedSideEffectImports`. Target ES2023 with bundler module resolution.

## ESLint & Prettier

- Flat config (`eslint.config.js`)
- `eslint-plugin-perfectionist` — imports and object properties sorted ascending/lexical
- Prettier: double quotes, semicolons, 2-space indent, trailing commas everywhere

## Panda CSS Notes

Config in `panda.config.mjs`. After modifying the config, run `npm run prepare` to regenerate `styled-system/`. Never manually edit files in `styled-system/`.

Design tokens: semantic color tokens for light/dark, typography tokens (Inter for body, Space Grotesk for headings), custom `scrollable` pattern.

## Deployment

Push to `master` → GitHub Actions builds and deploys to GitHub Pages automatically.

Workflow: `npm ci` → `npm run build` → upload `dist/` → deploy to Pages.
